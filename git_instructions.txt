# Инструкции по работе с Git

## Основные команды для работы с GitHub

### 1. Начальная настройка Git
```bash
# Установка имени пользователя
git config --global user.name "Ваше имя"

# Установка email
git config --global user.email "ваш.email@example.com"
```

### 2. Работа с репозиторием

#### Создание нового репозитория
```bash
# Инициализация Git в папке проекта
git init

# Добавление удаленного репозитория
git remote add origin https://github.com/username/repository.git
```

#### Первая отправка кода
```bash
# Добавление всех файлов
git add .

# Создание первого коммита
git commit -m "Initial commit"

# Создание ветки main
git branch -M main

# Отправка кода в GitHub
git push -u origin main
```

### 3. Обновление кода

#### Отправка изменений
```bash
# Добавление измененных файлов
git add .

# Создание коммита
git commit -m "Описание изменений"

# Отправка изменений в GitHub
git push
```

#### Получение изменений
```bash
# Получение изменений с GitHub
git pull
```

### 4. Работа с ветками

#### Создание новой ветки
```bash
# Создание ветки
git checkout -b feature/название-функции

# Переключение между ветками
git checkout main
git checkout feature/название-функции
```

#### Слияние веток
```bash
# Переключение на основную ветку
git checkout main

# Слияние ветки
git merge feature/название-функции
```

### 5. Откат изменений

#### Откат последнего коммита
```bash
# Откат последнего коммита (сохраняет изменения в рабочей директории)
git reset --soft HEAD~1

# Откат последнего коммита (удаляет изменения)
git reset --hard HEAD~1
```

#### Откат к определенному коммиту
```bash
# Просмотр истории коммитов
git log

# Откат к определенному коммиту
git reset --hard <hash-коммита>
```

#### Откат изменений в файле
```bash
# Откат изменений в конкретном файле
git checkout -- путь/к/файлу
```

### 6. Полезные команды

#### Просмотр статуса
```bash
# Просмотр состояния репозитория
git status

# Просмотр истории коммитов
git log

# Просмотр изменений в файлах
git diff
```

#### Очистка
```bash
# Удаление неотслеживаемых файлов
git clean -fd

# Отмена индексации файлов
git reset
```

## Важные замечания

1. Всегда делайте коммиты с понятными сообщениями
2. Перед отправкой изменений проверяйте статус репозитория (git status)
3. Регулярно делайте pull, чтобы получать изменения от других разработчиков
4. Создавайте отдельные ветки для новых функций
5. Будьте осторожны с командами reset --hard, они удаляют изменения безвозвратно
